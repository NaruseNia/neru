# neru スクリプト言語 要件定義書

## 概要

**neru** は、ノベルゲームエンジン **nerune-engine** 向けのスクリプト言語である。
シナリオライターとプログラマーの両方が快適に開発できるよう、2層構造の言語設計を採用する。

- **実装言語**: Zig
- **ライセンス**: MIT
- **エンジン**: nerune-engine（wgpu ベース、クロスプラットフォーム）

---

## 1. 言語設計

### 1.1 2層アーキテクチャ

| 層 | 対象 | 特徴 |
|---|---|---|
| シナリオ層 | ライター・ディレクター | マークダウン拡張構文。プレーンテキストがそのままセリフになる |
| ロジック層 | プログラマー | neru 独自のスクリプト言語。ゲームロジックを記述 |

### 1.2 シナリオ層構文（マークダウン拡張）

- プレーンテキストがそのままセリフ（テキスト表示命令）になる
- `@` プレフィックスで命令を記述
- `{variable}` で変数展開
- `#label` でラベル定義
- `#choice` で選択肢

```
@speaker 太郎
@bg 校庭.png --fade 500

「こんにちは、私の名前は{name}です」
「今日はいい天気だね」

#choice
  - 「そうだね」 -> good_route
  - 「そうかな？」 -> bad_route
```

### 1.3 ロジック層構文（独自言語）

ミドルレベルの言語機能を提供する:

- **変数**: `let` による宣言
- **制御構文**: `if` / `else` / `for` / `while`
- **関数**: `fn` による定義、引数・戻り値あり
- **データ構造**: 配列（`[]`）、マップ（`{}`）
- **型システム**: 動的型付け

```
fn check_affinity(char: str) -> int {
  return state.affinity[char]
}

if check_affinity("taro") >= 50 {
  goto true_end
} else {
  goto normal_end
}
```

### 1.4 層間連携

シナリオファイル内にインラインでロジックを埋め込み可能:

- `@call function()` で関数呼び出し
- `@if` / `@else` / `@end` でシナリオ内条件分岐
- `@eval expression` で式評価
- 簡単な条件分岐はシナリオ層だけで完結できる

```
@speaker 太郎
「アイテムをあげるよ」

@call give_item("sword")

@if state.hp > 50
  「元気そうだね」
@else
  「大丈夫？」
@end
```

## 2. 実行モデル

### 2.1 バイトコードVM（メイン）

- .neru ファイルをバイトコードにコンパイル
- 専用VMで実行
- Zig との相性が良く、パフォーマンスと実装のバランスが取れる

### 2.2 インタプリタモード（補助）

- 開発時の迅速なイテレーション用
- AST を直接ウォークして実行
- ホットリロード等の開発支援に活用

---

## 3. コア機能

### 3.1 テキスト表示・選択肢

- メッセージ表示（自動的にウェイト / クリック待ち）
- キャラクター名表示（`@speaker`）
- 選択肢分岐（`#choice`）
- テキスト内変数展開（`{variable}`）

### 3.2 演出制御

- 背景切替（`@bg`）
- 立ち絵表示/非表示（`@show` / `@hide`）
- BGM / SE 制御（`@bgm` / `@se`）
- トランジション / フェード効果

### 3.3 セーブ / ロード

- グローバル state の自動永続化
- スクリプトからセーブポイントを制御可能
- セーブスロット管理

### 3.4 バックログ / 履歴

- 既読管理
- バックログジャンプ
- CG 回収率トラッキング

---

## 4. シーン管理

### 4.1 ラベル + ジャンプ

- ファイル内ラベル: `# label_name`
- ファイル内ジャンプ: `@goto label_name`
- ファイル間ジャンプ: `@jump chapter1.neru#start`

### 4.2 チャプター / シーン単位

- ファイル単位でのチャプター管理もサポート
- ラベルとチャプターの両方を組み合わせて使用可能

---

## 5. 状態管理

### 5.1 グローバル state オブジェクト

- 組み込みの `state` オブジェクトにドットアクセスで値を格納
- セーブ / ロード時に自動永続化
- シナリオ層・ロジック層の両方からアクセス可能

```
// ロジック層
state.affinity.taro = 0
state.flags.met_taro = true
state.inventory = ["key"]

// シナリオ層
@if state.flags.met_taro
  「また会ったね」
@end
```

---

## 6. マクロ

繰り返しパターンをマクロとして定義・再利用:

```
@macro enter(name, emotion)
  @speaker {name}
  @show {name} {emotion} --fade 300
@end

@enter "太郎" "smile"
「こんにちは！」
```

## 7. 国際化（i18n）

- 初期リリースから対応
- テキストキー方式またはロケールファイル分離
- 言語切り替え機構を言語レベルでサポート

---

## 8. エンジン連携

### 8.1 疎結合設計

- neru は汎用的な中間表現（バイトコード + 命令セット）を出力
- nerune-engine がそれを解釈・実行
- エンジン差し替えが可能な設計

### 8.2 nerune-engine

- wgpu ベースのクロスプラットフォームレンダリング
- neru VM をホストし、スクリプトの命令を描画・音声に変換

---

## 9. エラーハンドリング

- **コンパイル時**: 構文エラー、未定義ラベル、未定義変数等を可能な限り検出
- **実行時**: 行番号付きエラーレポート
- コンパイル時に検出できるエラーを最大化する設計

---

## 10. ファイル構成

### 10.1 拡張子

- シナリオファイル: `.neru` — マークダウン拡張構文（インラインロジック `@if`, `@call` 等を含む）
- ロジックファイル: `.nerul` — 純粋なロジック記述（`fn`, `let`, `if` 等）
- パーサーは拡張子でモードを確定する（ヒューリスティック不要）

### 10.2 プロジェクト構成

```
my_game/
  neru.toml           # プロジェクト設定
  scripts/            # シナリオファイル
    prologue.neru
    chapter1.neru
  logic/              # ロジックファイル
    game.nerul
    items.nerul
  assets/             # 画像・音声等
    bg/
    character/
    bgm/
    se/
```

### 10.3 プロジェクト設定 (neru.toml)

```toml
[project]
name = "my_game"
version = "1.0.0"
entry = "scripts/prologue.neru"

[assets]
path = "assets/"

[build]
output = "dist/"
```

## 11. 開発者ツール

| ツール | 説明 |
|---|---|
| `neru compile` | .neru → バイトコードコンパイル |
| `neru run` | スクリプト実行（インタプリタモード） |
| `neru build` | プロジェクト全体のビルド・パッケージ化 |
| LSP | エディタ補完・シンタックスハイライト・エラー表示 |
| デバッガー | ステップ実行・ブレークポイント・変数ウォッチ |

## 12. 実装ロードマップ

| Phase | 内容 |
|---|---|
| **Phase 1** | レキサー・パーサー・バイトコードコンパイラ・基本VM |
| **Phase 2** | シナリオ層（テキスト表示・選択肢・演出命令） |
| **Phase 3** | ロジック層（変数・関数・ループ・配列/マップ） |
| **Phase 4** | 統合（state 管理・セーブ/ロード・マクロ・i18n） |
| **Phase 5** | ツール（CLI・LSP・デバッガー） |
